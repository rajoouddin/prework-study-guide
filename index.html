HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The title tag is the only element from the head element that a user will see
  • HTML is the structure (the frame of a house) to a webpage

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • We link the stylesheet to the html document and goes after the last meta tag
  • To style a class selector we need to add a period before the class name in the stylesheet

Git

  • Git is a version control software
  • Developers use it to pull information from repos held on things like GitHub so they can work on things locally
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: this will pull any changes from the repo to ensure your local copy is up to date
  • git checkout branch-name: switches the branch to the one you state
  • git add -A: adds all changes to the staging area
  • git commit -m "commit-message": used to save any changes held in staging and assigns is a unique key
  • git push origin branch-name: this will push any commits made to the branch you have finished working on to the repo

JavaScript

  • Variable are used to store data types as values
  • The JS file is linked to the bottom of the HTML file (within the body element) to allow for the page to load before any scripts run
  • JS code is read from top to bottom and order counts
  • Conditional statements such as if statements allows us to interrupt control flow
  • Arrays can store multiple items and can be different data types in a single variable
